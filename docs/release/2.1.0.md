## v2.1.0

Minor release - Adds a newsletter signup modal triggered from the Blog nav link, a reusable Modal component, and various
code quality and accessibility improvements.

---

### New Components

- **`Modal`** — Reusable portal-based modal with `role="dialog"`, `aria-modal`, focus trap (Tab/Shift+Tab), Escape and
  backdrop click to close, body scroll lock, focus restoration on close, and entrance animations (backdrop fade-in, card
  slide-up).
- **`NewsletterModal`** — Blog "Coming Soon" email capture modal wrapping `Modal`, with floating label email input,
  inline validation, server-side error/warning feedback, and animated success view.

### New Hooks

- **`useNewsletterForm`** — Form state management for the newsletter modal with email validation,
  `POST /api/subscribers` integration, 409 duplicate handling (inline warning), auto-reset from error state, and full
  `reset()` on modal close.

### Features

- **Blog nav → Newsletter modal** — The BLOG header link intercepts navigation and opens the newsletter signup modal
  instead of routing to `/blog`.
- **Subscriber duplicate detection** — `addSubscriber` checks Resend contacts before creating, returning a 409 with
  "Email already subscribed" for existing contacts.
- **Analytics events** — Added `blog-button-clicked`, `newsletter-subscribe-attempted`, and `newsletter-subscribed`
  event tracking via Umami.

### Accessibility

- Modal: focus trap, Escape to close, backdrop close, programmatic focus on open, focus restoration on close.
- Newsletter success view uses semantic `<output>` element (implicit `role="status"`).
- Contact `SubmitButton` now has `aria-label` ("Message sent" / "Send failed") when rendering icon-only states.
- Modal backdrop uses `role="presentation"` with `onKeyDown` handler.

### Code Quality

- Email regex hardened against ReDoS — backtracking-safe domain validation in both `useContactForm` and
  `useNewsletterForm`.
- Analytics payloads sanitized — `contact-attempted` now tracks only metadata (field presence, message length) instead
  of raw user content.
- Newsletter `track` event fires only after successful subscription, not before the request.
- Resend `contacts.get` error handling — distinguishes `not_found` (proceed to create) from real API failures.
- Removed deprecated `React.FormEvent` usage.

### Testing

- Modal: focus trap, Escape, backdrop click, scroll lock, focus restoration.
- Newsletter modal: success flow, 409 duplicate, server error, form reset on close.
- `useNewsletterForm` hook: validation, API responses, state transitions.
- `emails-sender`: subscriber creation, duplicate check, quota handling, get error handling.
